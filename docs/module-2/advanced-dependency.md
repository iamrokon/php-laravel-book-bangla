# Episode 7: Advanced Dependency Management

কম্পোজারের মাধ্যমে প্যাকেজ ইন্সটল করলে `composer.json` ফাইল তৈরি হওয়ার সাথে সাথে `composer.lock` নামে আরেকটি ফাইল তৈরি হয়। আমরা যখন `composer install` কমান্ড রান করাই, তখন আমাদের প্রজেক্টের প্যাকেজগুলোর ঠিক কোন ভার্সনগুলো (exact version) ব্যবহার করা হচ্ছে, তা এই `composer.lock` ফাইলে রেকর্ড করা হয়। 

এর সুবিধা হলো, অন্য কেউ এই প্রজেক্ট ক্লোন করে ইন্সটল করলে (বা অন্য পিসিতে নিলে), সেখানেও প্যাকেজগুলোর ঠিক ওই ভার্সনগুলোই ইন্সটল হবে, ফলে ভার্সন কনফ্লিক্টের কোনো ঝামেলা থাকে না। 

### Development vs Production Dependencies
কিছু প্যাকেজ আমাদের শুধু ডেভেলপমেন্ট এনভায়রনমেন্টে (development environment) লাগে, প্রোডাকশন (production) এনভায়রনমেন্টে লাগে না (যেমন: টেস্টিং প্যাকেজ, ডিবাগিং টুল)। এদেরকে আমরা `composer.json` ফাইলে `require-dev` এর মধ্যে রাখতে পারি। 

এজন্য আমাদের প্যাকেজ ইন্সটলের সময় `--dev` ফ্ল্যাগ দিয়ে কমান্ড চালাতে হবে। যেমন:

```bash
composer require nahid/jsonq --dev
```

এরপর `composer.json` ফাইলের আকার হবে এরকম:

```json
{
    "require": {
        "fakerphp/faker": "^1.23"
    },
    "require-dev": {
        "nahid/jsonq": "^6.0"
    }
}
```

যেহেতু প্রোডাকশন সার্ভারে আমাদের ডেভ ডিপেন্ডেন্সিগুলো দরকার নেই, তাই সেখানে প্যাকেজ ইন্সটল করার জন্য আমরা নিচের কমান্ডটি রান করব:

```bash
composer install --no-dev
```

### Useful Composer Commands
কম্পোজারে অ্যাভেইলঅ্যাবল (available) সব কমান্ড দেখার জন্য শুধু `composer` লিখে `Enter` চাপলেই হবে:

```bash
composer
```

এছাড়া কম্পোজারে আর কী কী কমান্ড রয়েছে, তা দেখার জন্য আমরা [কম্পোজারের অফিসিয়াল সাইটের ডকুমেন্টেশন](https://getcomposer.org/doc/) থেকে দেখে নিতে পারি।
